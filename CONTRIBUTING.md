# Contributing

Thank you for helping improve Awesome Generative AI for Material Discovery.

## Before submitting

- Make sure the work is directly relevant to generative AI for materials discovery.
- Search the README to avoid duplicate entries and unnecessary cross-listing.
- Prefer the publisher, proceedings, OpenReview, DOI, or arXiv page as the title link.
- Link to official code, datasets, documentation, demos, and leaderboards whenever available.
- Use the work's first public release date in `YYYY.MM.DD` format.
- Keep descriptions factual and avoid promotional language.

## Choosing a section

- **Algorithms**: the primary contribution is a model, representation, training method, or autonomous discovery workflow.
- **Surveys**: reviews, perspectives, tutorials, or broad capability assessments.
- **Datasets**: reusable, downloadable data is released.
- **Benchmarks**: explicit evaluation tasks, protocols, metrics, or leaderboards are provided.
- **Toolkits**: reusable multi-model software, data pipelines, or hosted evaluation services are provided.

Cross-list a work only when it makes independently reusable contributions to more than one resource type.

## Entry format

Add entries to the end of the appropriate chronological group using the existing table format:

```markdown
| [**Full paper title**](OFFICIAL_PAPER_URL) | Venue YEAR | YYYY.MM.DD | [GitHub](OFFICIAL_CODE_URL) | [Demo](OFFICIAL_DEMO_URL) |
```

Use `-` when a column has no verified link. If an official GitHub repository exists, add its stars badge before the paper title, following nearby entries.

## Pull requests

Keep each pull request focused. Complete the pull request checklist, briefly explain why the entry belongs in the chosen section, and do not reformat unrelated rows.
